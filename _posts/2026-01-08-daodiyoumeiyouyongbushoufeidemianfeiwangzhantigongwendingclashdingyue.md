<p><strong>Q1: 为什么导入订阅后显示“Empty”或没有节点？</strong></p>
<p>这通常是因为订阅链接已被墙，或者链接格式不兼容。尝试在浏览器中直接访问该链接，如果无法打开，说明需要挂代理才能更新订阅，这就陷入了“鸡生蛋”的死循环。解决方法是先找一个临时的可用节点（如通过 GitHub 找一个单独的节点字符串）复制到剪贴板导入，连通后再去更新大的订阅列表。</p>

<p><strong>Q2: 节点显示连通（绿色延迟），但无法访问网页？</strong></p>
<p>这大概率是系统时间不同步或 DNS 污染导致的。V2Ray 和 Clash 等协议对时间同步要求极高，系统时间与服务器时间误差超过 90 秒就会导致认证失败。此外，可以尝试在命令行清理 DNS 缓存：</p>

<code>ipconfig /flushdns</code>

<p><strong>Q3: 如何快速测试本地配置是否生效？</strong></p>
<p>不要只依赖浏览器的反馈，使用命令行工具 curl 可以更直观地看到连接状态和头部信息。在终端中输入以下命令，查看是否返回 HTTP 200 或 301 状态码：</p>

<code>curl -I https://www.google.com</code>
