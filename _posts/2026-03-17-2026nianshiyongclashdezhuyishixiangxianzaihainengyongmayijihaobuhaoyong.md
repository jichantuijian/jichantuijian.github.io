<h3>系统环境兼容性与 DNS 泄露的使用Clash的注意事项</h3>

<p>在 Windows 或 macOS 系统中使用时，Clash 的工作模式（系统代理模式 vs TUN 模式）对网络栈的影响完全不同。<strong>使用Clash的注意事项</strong>中，关于 DNS 污染和泄露的配置往往被用户忽略。如果 DNS 配置不当，即使节点处于连接状态，部分网站依然无法访问，或者会出现严重的解析延迟。</p>

<ul>
    <li><strong>TUN 模式的优势：</strong> 能够接管系统层面的所有流量，包括不支持系统代理的软件（如 UWP 应用、部分网游客户端）。</li>
    <li><strong>DNS 策略配置：</strong> 建议在 <code>dns</code> 字段下开启 <code>fake-ip</code> 模式，并配置合理的 <code>nameserver</code> 和 <code>fallback</code>。例如，使用阿里 DNS (223.5.5.5) 作为基础解析，配合 Cloudflare DNS (1.1.1.1) 进行加密解析。</li>
    <li><strong>IPv6 兼容性：</strong> 许多国内 ISP 已默认开启 IPv6，若 Clash 规则中未对 IPv6 流量进行合理分流，可能会导致流量绕过代理直连，从而产生连接中断。</li>
</ul>

<h3>故障排查常见问题的使用Clash的注意事项</h3>

<p>在日常使用中，用户经常会遇到节点全红或订阅无法更新的情况。掌握基础的排查逻辑是<strong>使用Clash的注意事项</strong>中不可或缺的一部分。以下是几个典型问题的处理思路：</p>

<p><code>为什么订阅链接更新失败，提示 Remote DNS Resolve Failed？</code>
这通常是因为当前系统 DNS 无法解析订阅服务器的域名。建议切换至手机热点或手动在 hosts 文件中添加订阅服务器的 IP 指向。此外，检查是否开启了系统代理，有时“代理循环”也会导致更新失败。</p>

<p><code>为什么节点延迟显示正常，但无法打开网页？</code>
这种情况多见于加密协议不匹配或系统时间不同步。Clash 依赖精确的系统时间（尤其是使用 Trojan 或 V2Ray 订阅时），如果时间偏差超过 90 秒，握手将会失败。此外，检查规则集中是否将该域名误判为 <code>REJECT</code>。</p>

<p><code>Shadowrocket 订阅和 Clash 订阅可以通用吗？</code>
两者格式不通用。Shadowrocket（小火箭）通常使用 Base64 编码的链接，而 Clash 使用 YAML 格式。<strong>使用Clash的注意事项</strong>提醒用户，必须通过后端转换器或客户端自带的导入功能进行格式转换，否则会报 <code>Invalid Config</code> 错误。</p>

<p><code>如何解决部分内网应用（如钉钉、微信）在开启代理后无法连接？</code>
这是典型的路由分流问题。应在配置文件中的 <code>skip-proxy</code> 或 <code>bypass</code> 列表中加入内网 CIDR 段（如 192.168.0.0/16）以及国内常用服务的域名后缀，确保这部分流量不经过代理内核。</p>
